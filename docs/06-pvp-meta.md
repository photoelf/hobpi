# 06. PvP, кланы и Telegram-специфика

## 1. Асинхронный PvP («разборка»)

Ключевая механика, наследие «Бойцовского клуба». Модель — **асинхронная**: не нужно ловить
соперника онлайн.

### Как работает

1. У каждого игрока есть **оборонительный снапшот**: слепок героя + бригады + артефактов,
   обновляемый автоматически при каждом изменении армии.
2. Атакующий открывает вкладку «Арена» → видит список из 5 соперников, подобранных по рейтингу.
3. Нажимает «Наехать» → сервер:
   - генерирует seed,
   - симулирует полный бой: атакующий под управлением ИИ **или** игрока (см. ниже),
   - записывает реплей и результат, начисляет рейтинг и трофеи,
   - шлёт защищавшемуся Telegram-уведомление.
4. Обе стороны могут посмотреть реплей и расшарить его в чат.

### Два режима атаки

| Режим | Описание | Стоимость |
|---|---|---|
| **Разборка** (ручная) | Атакующий играет бой интерактивно против ИИ обороны. Максимальный контроль. | 2 топлива |
| **Автобой** | Обе стороны ведёт ИИ, результат мгновенный. Для фарма и когда некогда. | 2 топлива |

Обороняющийся **всегда** под ИИ. Это единственный честный вариант для асинхронного формата.

### Рейтинг

Elo с K=32, стартовый рейтинг 1000:

```
ожидание = 1 / (1 + 10^((rB - rA) / 400))
rA' = rA + K * (результат - ожидание)
```

Защита от абуза:
- нельзя атаковать одного и того же игрока чаще 1 раза в 6 часов,
- нельзя атаковать игрока с рейтингом ниже на 250+ (нет награды),
- «щит новичка»: первые 48 часов и до 5 уровня игрока нельзя атаковать,
- «щит после поражения»: 30 минут неуязвимости после проигранной обороны.

### Трофеи

Победа в PvP: `налик = 5% казны проигравшего (макс. 3000)`, `+2 влияния`, `+XP`.
Потери армии **реальные** для обеих сторон — это главная цена разборки и главный тормоз спама.
Обороняющийся при поражении теряет только 50% павших (остальные «отлежались»).

## 2. Матчмейкинг

Подбор по формуле силы `power = 0.6 * ai_value(армия) + 0.4 * (рейтинг)`:
- 5 кандидатов в окне ±15% power,
- если кандидатов мало — окно расширяется шагами по 10% до ±50%,
- в списке всегда 1 «слабый» (лёгкая цель, мало наград) и 1 «сильный» (риск/куш).

## 3. Кланы (v0.3)

- Клан = 5–30 игроков. Создание: ур. 10 + 20 000 налика.
- **Клановая казна**: взносы игроков, тратится на клановые апгрейды (глобальные баффы всем).
- **Клановые войны**: раз в неделю два клана матчатся по суммарной силе; за 72 часа надо
  набрать больше очков атак по членам вражеского клана. Победитель получает контроль над
  «ключевой точкой» района на следующую неделю.
- **Ключевые точки** (аналог городов HoMM3): Порт, Апрашка, Крестовский, Бизнес-центр,
  Промзона. Дают всему клану пассивный бонус (+% дохода / прироста / влияния).

## 4. Telegram-специфика

### Аутентификация
`Telegram.WebApp.initData` → HMAC-SHA256 проверка на сервере по схеме Telegram
(`secret = HMAC_SHA256("WebAppData", BOT_TOKEN)`, затем проверка `hash` над отсортированным
data-check-string). Срок годности initData — 24 часа. Никаких паролей.

### Уведомления (через бота)
| Событие | Текст |
|---|---|
| На тебя наехали | «🚨 {name} наехал на твой район. Итог: {результат}. [Смотреть реплей]» |
| Топливо полное | «⛽ Мерин заправлен, 10/10. Пора на разборку.» |
| Казна переполнена | «💰 Касса ломится. Забери, пока не отжали.» |
| Недельное событие | «📅 Началась {событие}.» |
| Клановая война | «⚔️ Война с {клан}. Осталось {N} часов.» |

Частота: не больше 3 пушей в сутки, с настройкой отключения по категориям.

### Виральные петли
1. **Шаринг реплея**: кнопка «Отправить в чат» → `switchInlineQuery` с deep-link на бой.
   Получатель открывает мини-апп сразу на экране реплея.
2. **Вызов из чата**: инлайн-режим бота `@bot вызов` → карточка «Наехать на {name}».
3. **Реферал**: `start_param` = id пригласившего; оба получают по 2 000 налика и артефакт
   за достижение приглашённым 5 уровня.
4. **Клановые чаты**: бот в групповом чате постит итоги клановой войны и топ недели.

### Ограничения платформы, которые учитываем
- Мини-апп открывается в WebView: без тяжёлых бандлов (цель — **< 300 КБ gzip** на первый экран).
- Нет надёжного фонового исполнения → всё начисление ленивое, на стороне сервера.
- Haptic feedback и тема оформления берутся из `Telegram.WebApp` (`themeParams`, `HapticFeedback`).
- Кнопка «Назад» — системная `BackButton`, не рисуем свою.
- Свайп-закрытие отключаем на экране боя (`disableVerticalSwipes`).

## 5. Античит

Поскольку бой считает **сервер**, клиент не может подделать результат. Дополнительно:
- rate-limit на все мутирующие эндпоинты,
- валидация каждого действия боя на сервере (легальность хода, очередь, ресурсы),
- бой имеет TTL (30 минут) — брошенные бои автоматически проигрываются,
- лимит боёв в час на аккаунт,
- детект мультиаккинга по паттерну «донор→реципиент» в PvP-трофеях (v0.4).
