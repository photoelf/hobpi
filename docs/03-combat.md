# 03. Боевая система

Реализация: `packages/engine`. Движок **детерминированный** и **изоморфный** — один и тот же код
крутится на сервере (авторитетная симуляция) и на клиенте (проигрывание реплея).

## 1. Поле

- Сетка **12 колонок × 9 рядов** (квадратные клетки, не гексы — читаемее на телефоне).
- Соседство — 8-связное (включая диагонали), дальность = чебышевское расстояние.
- Левая сторона (колонки 0) — атакующий, правая (колонка 11) — обороняющийся.
- Препятствий на поле в MVP нет (v0.4: «мусорные баки», «гаражи» как блокеры).

## 2. Стек

Стек = `{ unitId, count, hpTop, x, y, side }`.

- `count` — сколько бойцов живо.
- `hpTop` — текущие ХП **верхнего** (принимающего урон) бойца.
- Эффективное ХП стека = `(count - 1) * hp + hpTop`.
- До 7 стеков на сторону, стартовая расстановка — вертикальная колонка по центру ряда.

## 3. Очередь хода (инициатива)

Round-based, как в HoMM3:

1. В начале раунда формируется очередь всех живых стеков, отсортированная по `speed` **убыв.**,
   тай-брейк — `initiative`, затем сторона (атакующий раньше), затем индекс стека.
2. Стек, выбравший **Ожидание**, уходит в «хвост очереди» текущего раунда: ожидавшие ходят
   после всех неожидавших, между собой — по **возрастанию** скорости (правило HoMM3).
3. Раунд заканчивается, когда все походили. Раунд-лимит: **30**, дальше — ничья по эффективным ХП.

## 4. Действия

| Действие | Описание |
|---|---|
| `MOVE` | Переместиться в достижимую клетку (BFS по свободным клеткам, дальность = `speed`). Для `FLYER`/`Проезд` — прямое чебышевское расстояние, важна только свободность конечной клетки. |
| `ATTACK` | Подойти к цели (если нужно) и ударить в ближнем бою. |
| `SHOOT` | Выстрел на любую дистанцию (нужны `shots > 0` и отсутствие врага в соседних клетках). |
| `DEFEND` | +30% к защите до следующего хода, ход завершается. |
| `WAIT` | Отложить ход (см. п.3). Можно 1 раз за раунд. |
| `CAST` | Приём героя. Не занимает ход стека, доступен 1 раз за раунд, тратит «связи». |

## 5. Формула урона

Прямой перенос HoMM3:

```
base       = rand(minDmg, maxDmg) * count
A          = attack(атакующего) + герой.Сила + бонусы
D          = defense(цели)      + герой.Броня + бонусы + (защищается ? 30% : 0)

если A > D:  mult = 1 + 0.05 * (A - D)      , clamp ≤ 4.0   (макс +300%)
если A < D:  mult = 1 - 0.025 * (D - A)     , clamp ≥ 0.3   (макс −70%)
иначе:       mult = 1

dmg = floor(base * mult * ситуативные_множители)
```

Ситуативные множители:

- Стрелок бьёт в ближнем бою: **×0.5** (кроме фракционного бонуса Айтишников).
- Стрелок стреляет дальше 8 клеток: **×0.5** (штраф дальности).
- Ответный удар: **×1.0** (без штрафа, как в HoMM3).
- Удача сработала: **×2.0**.
- Фракционные и способностные модификаторы — аддитивно к множителю.

## 6. Применение урона к стеку

```
kills   = 0
rem     = dmg
if rem >= hpTop: rem -= hpTop; kills = 1; hpTop = hp
kills  += floor(rem / hp)
hpTop  -= rem % hp   (с переносом)
count  -= kills       (не ниже 0)
```

## 7. Ответный удар (ответка)

- Цель отвечает **1 раз за раунд**, если жива, не является стрелком-в-упор-без-права,
  и атакующий не имеет `NO_RETALIATION`.
- Счётчик ответок сбрасывается в начале раунда.
- Ответка не наносит ответку.

## 8. Мораль и удача

| Стат | Диапазон | Эффект |
|---|---|---|
| Мораль | −3 … +3 | Положительная: шанс `мораль × 4.2%` на **дополнительный ход** сразу после текущего. Отрицательная: шанс `|мораль| × 4.2%` **пропустить ход**. |
| Удача | −3 … +3 | Положительная: шанс `удача × 4.2%` на **двойной урон**. Отрицательная: шанс на **половинный урон**. |

Источники: фракционный бонус, аура юнита, артефакты, приёмы героя, навыки.

## 9. Свойства бойцов (traits)

Реализованные флаги движка:

| Флаг | Эффект |
|---|---|
| `SHOOTER` | Может стрелять; штраф ×0.5 в ближнем бою и на дальности >8 |
| `FLYER` | Игнорирует занятость промежуточных клеток при движении («Проезд») |
| `NO_RETALIATION` | Цель не отвечает |
| `DOUBLE_STRIKE` | Бьёт дважды за одну атаку |
| `TAUNT` | Провокация: соседние враги обязаны атаковать этот стек, если он в зоне досягаемости |
| `AURA_MORALE` | +1 (или +2) морали союзникам, пока жив |
| `AURA_LUCK` | +1 удачи союзникам, пока жив |
| `HEAL` | Вместо атаки лечит соседний союзный стек на % макс. ХП |
| `RESURRECT` | Один раз за бой воскрешает бойцов в союзном стеке |
| `SLOW_ON_HIT` | −2 к скорости цели на 1 раунд |
| `ARMOR_BREAK` | −40% защиты цели до конца боя |
| `SHIELD_NEIGHBORS` | −25% урона по соседним союзным стекам |
| `DEBUFF_ATTACK` | −2 к атаке цели на 2 раунда |
| `DEBUFF_DEFENSE` | −15% защиты цели на 2 раунда |
| `SPLASH_NEIGHBORS` | 50% урона по всем соседям цели |
| `BLINK` | «Рывок»: перемещение в любую свободную клетку поля |
| `STUN_ONCE` | Один раз за бой цель пропускает следующий ход |

## 10. Приёмы героя («магия»)

Герой не имеет ХП и не может быть убит. Раз в раунд он может применить один приём, потратив
«связи» (мана = `Связи × 10`). Сила эффектов масштабируется от стата **Авторитет**.

| Приём | Школа | Цена | Эффект |
|---|---|---|---|
| Адреналин | Бафф | 6 | +2 скорости, +1 мораль союзному стеку на 3 раунда |
| Общий сбор | Бафф | 9 | +2 морали **всей** бригаде на 2 раунда |
| Фонарик в глаза | Дебафф | 7 | −50% урона вражескому стрелку на 2 раунда |
| Облава | Контроль | 12 | Вражеский стек пропускает следующий ход |
| Проверка документов | Контроль | 8 | Снимает все баффы с вражеского стека |
| Запугивание | Дебафф | 9 | −3 атаки и −1 мораль вражескому стеку на 3 раунда |
| Санитары | Поддержка | 10 | Лечит `Авторитет × 25` ХП союзному стеку |
| Связь с больничкой | Поддержка | 14 | Телепорт союзного стека в любую свободную клетку |
| Наезд | Урон | 11 | Прямой урон `Авторитет × 20` вражескому стеку |

Приёмы открываются в здании «Сигарная комната» (аналог Гильдии магов) и ограничены его уровнем.

## 11. Условия окончания

- Все стеки одной стороны мертвы → победа другой.
- 30 раундов → победа по большему остатку эффективных ХП, при равенстве — ничья.
- Отступление (только PvE): бой прерывается, армия сохраняется, трофеев нет, топливо тратится.

## 12. Детерминизм и реплеи

- Единственный источник случайности — PRNG **mulberry32** с 32-битным seed.
- Seed выдаёт сервер и хранит в записи боя.
- Бой сериализуется как `{ seed, setup, actions[] }`. Полный лог событий (`BattleEvent[]`)
  восстанавливается прогоном движка → реплей весит сотни байт, а не мегабайты.
- Клиент воспроизводит реплей тем же движком → анимация всегда совпадает с расчётом сервера.

## 13. ИИ

Один ИИ обслуживает и нейтралов в PvE, и обороняющегося в асинхронном PvP.
Эвристика хода (жадная, с оценкой):

1. Если стек — стрелок и патроны есть и рядом нет врага → стрелять по цели с максимальным
   `score = ожидаемый_урон + бонус_за_добивание − риск_ответки`.
2. Если можно достать врага в этот ход → атаковать цель с максимальным `score`.
3. Если врага не достать → двигаться к ближайшему по «ценности» врагу (приоритет: вражеские
   стрелки и хилеры).
4. Хилеры лечат раненый союзный стек, если урон от лечения > урона от атаки.
5. Герой ИИ кастует приём, если его ожидаемая ценность > порога (урон/контроль по крупнейшему
   вражескому стеку), не раньше 2-го раунда.

Уровни сложности регулируются: глубиной оценки, вероятностью «ошибки» (случайного хода)
и доступом ИИ к приёмам героя.
