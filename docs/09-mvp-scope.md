# 09. Скоуп MVP (v0.1)

Цель MVP: **доказать, что тактический бой HoMM3-уровня работает и приятен на телефоне в Telegram,
и что асинхронный PvP между TG-аккаунтами создаёт возвраты.** Всё, что не служит этой цели,
вырезано.

## ✅ Что вошло

### Движок боя (`packages/engine`)
- Сетка 12×9, до 7 стеков на сторону.
- Очередь инициативы с правилом ожидания HoMM3.
- Действия: движение, атака, выстрел, защита, ожидание, приём героя.
- HoMM3-формула урона, ответные удары, мораль, удача.
- 17 свойств бойцов (стрелок, проезд, без ответки, двойной удар, провокация, ауры, лечение,
  воскрешение, замедление, пробой брони, прикрытие, дебаффы, сплэш, рывок, стан).
- 9 приёмов героя.
- Детерминированный PRNG, полная сериализация боя в `{seed, setup, actions}`.
- Эвристический ИИ для нейтралов и обороны в PvP.
- Оценка силы армии `ai_value` для матчмейкинга.

### Контент
- **4 фракции** × 7 тиров = **28 бойцов** с уникальными способностями.
- **3 класса героя**.
- **18 артефактов**, 8 слотов экипировки (включая слот «Ствол»).
- **13 зданий** и **6 типов точек дохода**.
- 6 недельных событий.

### Сервер (`apps/api`)
- Авторизация по Telegram `initData` (HMAC-SHA256) + dev-режим.
- Ленивое начисление дохода, прироста бойцов и топлива.
- Найм, постройка/апгрейд зданий и точек, экипировка артефактов.
- Прокачка героя (XP → уровень → статы).
- **PvE**: генерация нейтральных «дворов» по силе игрока, интерактивный бой.
- **PvP**: матчмейкинг, ручной бой и автобой, Elo-рейтинг, трофеи, реальные потери армии.
- Сохранение реплеев, лидерборд.

### Клиент (`apps/web`)
- Мобайл-фёрст тёмная тема в стиле 90-х, интеграция с темой Telegram.
- Онбординг: фракция → класс → кличка.
- 5 экранов: База, Бригада, Район, Арена, Топ + экран Героя.
- Экран боя: сетка на CSS Grid, очередь инициативы, подсветка досягаемости, панель приёмов.
- Проигрывание реплеев PvP.
- Haptic feedback, системная кнопка «Назад».

## ❌ Что сознательно НЕ вошло

| Не вошло | Почему |
|---|---|
| Остальные 5 фракций | 4 достаточно для проверки контр-пиков; каждая новая — это неделя баланса |
| Улучшенные версии бойцов (T1+ … T7+) | Удваивает контент, не добавляет нового опыта на этапе проверки гипотезы |
| Настоящая карта города с перемещением героя | Долгие сессии противоречат мобильному формату; в MVP «Район» = список целей |
| Кланы и клановые войны | Требуют критической массы игроков; бессмысленны на 100 тестерах |
| Кампании и нарратив | Дорогой контент, не проверяет ядро |
| Комбо-артефакты, Грааль-здание | End-game, до него в MVP не дойдут |
| Монетизация, Telegram Stars | Сначала retention, потом деньги |
| Барахолка / трейд между игроками | Открывает вектор мультиаккинга, нужен античит |
| Анимации спрайтов бойцов | Нет арта; MVP на типографике и иконках-эмодзи |
| Real-time PvP | Асинхрон — сознательный выбор формата (см. [06-pvp-meta.md](06-pvp-meta.md)) |

## Определение готовности MVP (Definition of Done)

- [x] Бой из 30 раундов симулируется детерминированно, тесты движка зелёные.
- [x] Реплей, восстановленный из `{seed, actions}`, совпадает с серверным расчётом бит-в-бит.
- [x] Игрок может пройти путь: онбординг → доход → найм → PvE-бой → PvP-разборка → рейтинг.
- [x] Сервер не доверяет клиенту ни одного игрового решения.
- [ ] Прогон с 20 живыми тестерами в течение недели без критических багов баланса.
- [ ] Бот-уведомления подключены к реальному `BOT_TOKEN`.

Последние два пункта требуют реального бота и живых игроков — они за пределами кода.
